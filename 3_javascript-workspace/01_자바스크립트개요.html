JavaScript Overview

1. 스크립트 언어

- 이미 만들어진 프로그램의 동작을 사용자의 요구에 맞게 수행되도록 해주는 용도
- 프로그램을 새로 만드는 언어가 아니라, 이미 만들어진 프로그램에 추가적인 동작을 덧씌워주는 언어



2. 자바스크립트

- 웹 브라우저에서 가장 많이, 기본적으로 사용되는 인터프리터 방식 스크립트 언어



3. 자바스크립트 장단점

장점

  • 간단한 코드 작성으로 초보자가 접하기 쉬운언어(단순한 구조, 원칙)
    ex) 접근제한자의 개념 없음, 변수 선언시 자료형 지정 필요 없음, 메소드 정의 시 반환형 정의 필요 없음
  • 컴파일 과정을 거치지 않기 때문에 실행 속도가 빠름

단점

  • 웹에 특화된 기술(내부적으로 제공되는 기능이 제한적)
  • HTML 내에 소스코드를 작성하게 되면 개발자 도구를 통해 소스코드가 그대로 노출되어 보안에 취약함


4. 자바스크립트 소스코드 작성 위치에 따른 종류

  1. 인라인 방식

    - HTML 요소의 이벤트 속성값으로 자바스크립트 코드를 작성하는 방식

  2. 내부 방식

    - HTML 문서 내의 <script> 태그 사이에 자바스크립트 코드를 작성하는 방식

  3. 외부 방식

    - 자바스크립트 코드를 별도의 파일로 작성하여 HTML 문서에서 불러와 사용하는 방식


인라인 방식

- 시작태그 내에 직적적으로 실행할 간단한 소스코드 작성해서 실행
- 주로 실행해야 하는 소스코드가 매우 소량일 경우 사용 

        <시작태그 이벤트속성명="자바스크립트 소스코드">
        </종료태그>


내부 방식

- HTML 문서 내에 <script> 태그 영역 안쪽에 함수 단위로 소스코드 작성해두고, 어떤 html 요소에 어떤 이벤트 발생 시 해당 함수를 호출함으로써 실행시킴
- script 태그는 head, body 태그 안쪽 아무데서나 작성 가능
