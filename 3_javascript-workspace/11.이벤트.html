event

1. event?

- web site를 이용하는 사용자에 의해 발생되는 사건 : 클릭, 더블클릭, 마우스 이동, 키보드 입력, 스크롤 등등

- 해당 이벤트가 발생했을 시 처리 방법에 대해 구현(=이벤트 처리)

- 이벤트 처리 시 고려 사항 1. event target : 이벤트가 발생한 요소(대상) 2. event type : 이벤트의 종류 3. event handler : 이벤트가 발생했을 때 실행할 함수



2. event 모델 종류

  1. 고전 이벤트 모델(기본 이벤트 모델)
  2. 인라인 이벤트 모델
  3. 표준 이벤트 모델


고전 이벤트 모델

- 요소 객체를 코드상으로 끌고 와서 해당 요소 객체의 이벤트 속성에 직접 접근 후 이벤트 핸들러 함수를 대입(연결) 해주는 방식 - 해당 이벤트 속성에 null을 대입하면 이벤트 핸들러 제거 가능



인라인 이벤트 모델

- 요소의 시작태그 쪽에 직접적으로 이벤트 속성을 제시 속성값으로 실행할 내용을 코드로 정의 - script 태그 내에 정의되어 있는 선언식 함수를 호출하는 방식을 더 선호 (인라인 이벤트 모델은 HTML과 JavaScript 코드가 뒤섞여서 가독성이 떨어지는 단점이 있기 때문)



표준 이벤트 모델

- 요소 객체의 이벤트 처리와 관련된 메서드를 이용해서 이벤트 핸들러를 연결하는 방식 - addEventListener() 메서드로 이벤트 핸들러 연결 - removeEventListener() 메서드로 이벤트 핸들러 제거 => 고전 이벤트 모델과 달리, 표준 이벤트 모델은 addEventListener() 메서드를 이용해서 이벤트 핸들러를 연결하기 때문에, 하나의 요소에 여러 개의 이벤트 핸들러를 연결할 수 있음



3. 현재 이벤트가 발생한 해당 요소객체에 접근하는 방법



4. 태그별 기본적으로 가지고 있는 이벤트 제거

태그 내에 기본적으로 설정되어 있는 이벤트를 제거하는 방법
: 이벤트 속성 내부에 return false; 코드를 작성해주면 해당 이벤트 제거 가능
