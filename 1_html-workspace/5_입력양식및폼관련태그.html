폼 관련 태그

입력 양식의 틀을 의미
form 태그로 항상 입력 양식들을 감싸줘야만 한다.
form 태그 내의 submit 버튼 클릭 시 해당 form 내에 작성된 사용자가 입력한 값들을 서버의 Controller 로 넘기면서 요청하는 역할 수행 

            [ 표현법 ]

            <form action="" method="">

                각종 input 태그를 조합하여 입력양식 화면 꾸리기

                submit (제출) 버튼

            </form>

            * action 속성 : 해당 form 내에 사용자가 입력한 값들을 전달하면서
                            요청할 서버의 경로를 제시
                            (즉, 어느 Controller 로 요청을 보낼건지
                             요청이 도달할 목적지를 기술하는 부분)

            * method 속성 : 요청 전송 방식을 지정하는 속성
                            요청 전송 방식에는 get (기본값) / post 방식이 있다.

            - get 방식 : 요청 시 사용자가 입력한 값들이 url 주소에 노출되는 방식 (검색 등)
            - post 방식 : 요청 시 사용자가 입력한 값들이 url 주소에 노출되지 않는 방식
                          주로 민감정보를 넘길 때 쓰는 것이 좋음 (로그인, 회원가입 등)

            form 태그 내에서 input 요소들을 통해 값들을 입력받고
            제출 (submit) 버튼 클릭 시
            action 에 지정한 Controller 로 method 에 작성한 요청 전송 방식으로
            사용자가 입력한 값들을 key=value&key=value&... 세트로 전달한다.
            이 때, 여러 key=value&key=value&... 를 "쿼리스트링" 이라고 부른다!!

라디오버튼과 체크박스 관련 input 태그의 타입

라디오버튼 (type="radio")

- 제시한 여러개의 값들 중에서 오로지 한개만 선택해야할 때 사용
- name 속성값을 동일하게 작성하여 라디오버튼끼리 묶으면 하나의 그룹으로 이어짐 (그 묶인 라디오버튼 중 하나만 체크되게끔)
- 제출 버튼 클릭 시 현재 선택된 (checked) value 값이 넘어감

성별 :

체크박스 (type="checkbox")

- 제시한 여러개의 값들 중에서 여러개를 선택 가능한 경우 사용
- name 속성값을 동일하게 작성하여 체크박스끼리 묶으면 하나의 그룹으로 이어짐 (그 묶인 체크박스들 중 여러개를 체크 가능하게끔)
- 제출 버튼 클릭 시 현재 선택된 (checked) value 값들이 넘어감

select 태그와 option 태그

- 사용자에게 직접 키보드로 값을 입력받는게 아니라 내가 제시한 여러 옵션들 중에서 드롭다운 형식으로 선택할 수 있도록 해줌
- select 태그로 여러개의 option 태그들을 묶어서 표현한다.
- select 태그에 name 속성을 한번만 적어서 키값을 정의해두고, 각 option 태그마다 value 속성값으로 넘길 값을 적어준다!!
- 제출 버튼 클릭 시 현재 선택된 (selected) value 값이 넘어감
(단, option 태그에 value 속성을 생략하면 시작태그와 종료태그 사이의 값이 value 로써 넘어간다) - 명시적으로 selected 속성을 지정해서 기본 선택값을 지정할 수 있다.
(명시적으로 selected 속성을 적지 않으면 기본적으로 맨 위의 항목이 기본 선택값이 된다)
